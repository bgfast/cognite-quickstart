# Atlas AI training slides — process and export

Markdown-driven slide deck for the [Atlas AI training agenda](atlas-ai-training-agenda.md). Built with [Slidev](https://sli.dev/); minimal content per slide and detailed speaker notes.

## Files

| File | Purpose |
|------|--------|
| `docs/atlas-ai-training-slides.md` | Source deck (edit this). Slidev uses `---` between slides and `<!-- ... -->` for speaker notes. |
| `docs/slides/atlas-ai-training-slides.pptx` | Exported PPTX (generated by `npm run export:pptx`). |
| `docs/global-top.vue` | Slidev global layer: styles content slides (gradient background, blue headings, accent bar). Cover slide is unchanged. |
| `slidev/package.json` | Slidev build/export scripts and dev deps: `@slidev/cli`, `@slidev/theme-default`, `playwright-chromium`. Run all npm commands from `slidev/`. |
| `docs/atlas-ai-training-agenda.md` | Training agenda; includes a short “Slides and export” section. |

## Prerequisites

- **Node.js** (LTS) and **npm**
- First-time export needs **playwright-chromium** (installed with `npm install`); Slidev uses it to render slides for PPTX/PDF.

## Process

### 1. Install dependencies

From the **`slidev/`** directory (Slidev build/export tooling):

```bash
cd slidev
npm install
```

Installs `slidev` and `playwright-chromium` (and any other devDependencies).

### 2. Present or edit slides

From **`slidev/`**:

```bash
npm run slides
```

Opens the deck in the browser with Slidev’s UI. Use **presenter mode** (e.g. via the nav or `/presenter`) to see speaker notes. Edit `docs/atlas-ai-training-slides.md` and save; the preview reloads.

### 3. Export to PPTX

From **`slidev/`**:

```bash
npm run export:pptx
```

- Slidev renders each slide and **embeds speaker notes** into the PPTX.
- **Output:** Slidev writes the file to **`docs/slides/atlas-ai-training-slides.pptx`**.
- Use the PPTX in PowerPoint or for importing into Google Slides (see below).

### 4. Export to PDF (optional)

From **`slidev/`**:

```bash
npm run export:pdf
```

Produces a PDF handout. Presenter notes are not shown in the PDF; use Slidev presenter view or the PPTX for notes.

## Using the PPTX in Google Slides

1. **Upload to Drive:** Upload the exported `.pptx` to Google Drive.
2. **Open with Google Slides:** Right-click the file → **Open with** → **Google Slides**, or open [Google Slides](https://slides.google.com) and open the file from Drive.
3. **Or import into an existing deck:** In Google Slides, **File → Import slides**. Choose **Upload** and select the PPTX. Pick which slides to import and click **Import slides**.

Speaker notes from the PPTX are preserved when you open or import the file in Google Slides.

## Slide format (for editors)

- **New slide:** `---` on a line by itself.
- **Speaker notes:** Put notes in an HTML comment after the slide content, e.g. `<!-- Your note here. -->`. These show in Slidev presenter view and in the exported PPTX.
- **First slide:** Optional frontmatter, e.g. `layout: cover` for the title slide.
- Keep slide **body** short (one idea, a few bullets); put detail in the **notes**.

## Troubleshooting

- **Export fails or hangs:** Ensure `npm install` completed and that `playwright-chromium` is installed. On first export, Slidev may download browser binaries.
- **Notes missing in PPTX:** Use the script `npm run export:pptx`; do not use a custom export path unless you know Slidev supports notes for it.
- **Deck not found:** Run all commands from the **`slidev/`** directory so that the relative paths to `../docs/atlas-ai-training-slides.md` and `../docs/slides/` resolve correctly.
