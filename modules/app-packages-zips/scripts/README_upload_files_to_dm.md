# Upload Files to Data Model (CDF)

## Overview

The `upload_files_to_dm.py` script uploads zip files to CDF Files API and links them to data model instances in the `app-packages` space. This is a workaround for the Cognite Toolkit limitation where it creates data model instances but doesn't upload the actual file content.

## What It Does

1. **Finds zip files** in the `downloads/` directory
2. **Creates data model instances** for each file in the `app-packages` space
3. **Uploads file content** to CDF Files API using `upload_content()` with `instance_id`
4. **Links files to instances** so they appear as `uploaded: True` in the Data Modeling API

## Why This Script Exists

The Cognite Toolkit has a limitation when using `space:` in `CogniteFile.yaml`:
- ✅ **Creates data model instances** (metadata)
- ❌ **Doesn't upload file content** to Files API
- ❌ **Results in `uploaded: False`** even when files exist

This script bridges that gap by manually uploading the files and linking them to the instances.

## Prerequisites

### 1. Environment Variables
Set up your CDF authentication environment variables:

```bash
export CDF_PROJECT="your-project"
export CDF_CLUSTER="your-cluster" 
export CDF_URL="https://your-cluster.cognitedata.com"
export IDP_CLIENT_ID="your-client-id"
export IDP_CLIENT_SECRET="your-client-secret"
export IDP_TENANT_ID="your-tenant-id"
export IDP_TOKEN_URL="https://login.microsoftonline.com/your-tenant-id/oauth2/v2.0/token"
```

### 2. Required Files
- Zip files must exist in `modules/app-packages-zips/downloads/`
- Files should be generated by `download_packages.py` script

### 3. CDF Setup
- `app-packages` space must exist
- `CogniteFile` view must be deployed
- Data model instances should be created (via toolkit deployment)

## Usage

### Basic Usage
```bash

python upload_files_to_dm.py
```

1. **Scans downloads directory** for `*.zip` files

3. **Creates data model instances** for each file
4. **Uploads files** to CDF Files API
5. **Links files to instances** using `instance_id`
6. **Reports success/failure** for each file

## Configuration

### File Locations
- **Downloads Directory**: `modules/app-packages-zips/downloads/`
- **Instance Space**: `app-packages`
- **View**: `cdf_cdm:CogniteFile/v1`

### File Naming
- Uses filename as `external_id` for data model instance
- Example: `cognite-quickstart-main.zip` → `external_id: cognite-quickstart-main.zip`

## Troubleshooting

### Common Issues

#### 1. Missing Environment Variables
```
❌ Missing required environment variables for CDF authentication
```
**Solution**: Set all required environment variables (see Prerequisites)

#### 2. File Not Found
```
❌ File not found: filename.zip
```
**Solution**: Run `download_packages.py` first to generate zip files

#### 3. CDF API Errors
```
❌ CDF API Error: [error details]
```
**Solution**: Check CDF permissions and space configuration

#### 4. Instance Already Exists
```
❌ CDF API Error: Instance already exists
```
**Solution**: Delete existing instances first or use different external_id

## Integration with Workflow

### 1. Download Phase
```bash
python download_packages.py  # Creates zip files in downloads/
```

### 2. Deploy Phase
```bash
cdf deploy --env=app-packages-zips  # Creates data model instances
```

### 3. Upload Phase
```bash
python upload_files_to_dm.py  # Uploads files and links to instances
```

### 4. Streamlit Phase
- Streamlit app discovers files via Data Modeling API
- Files show `uploaded: True`
- Users can download and deploy configurations

## Technical Details

### Data Model Instance Structure
```python
{
    "instanceType": "node",
    "space": "app-packages",
    "externalId": "filename.zip",
    "sources": [
        {
            "source": {
                "type": "view",
                "space": "cdf_cdm",
                "externalId": "CogniteFile",
                "version": "v1",
            },
            "properties": {
                "name": "filename.zip",
                "mimeType": "application/zip",
            }
        }
    ]
}
```

### File Upload Method
```python
client.files.upload_content(
    path=file_path,
    instance_id=NodeId(space, external_id)
)
```
